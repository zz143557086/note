1.为什么需要docker 容器虚拟化技术
 可以将配置环境打包成一个镜像文件 
 合作者可以直接使用镜像环境不需要再加配置
 可以达到跨平台的无缝接轨运作
 环境隔离的作用
 更小，更快，更简单
2.docker好处
 更快速的应用交互和部署
 更便捷的升级和扩缩容
 更简单的系统运维
 更高效的计算机资源利用
3.Docker 的三个关键要素是镜像（Image）、容器（Container）和仓库（Repository）
 可以通过 Dockerfile 来定义镜像的构建过程。镜像可以用来创建和部署容器。
 每个容器都是相互隔离的，具有自己的文件系统和进程空间。容器可以被启动、停止、删除等操作
 Docker 仓库用于存储和共享 Docker 镜像
 docker按照镜像生成的容器实例
4.docker run 镜像 
如果存在镜像直接运行 如果不存在 区docker hub拉取镜像有就运行 没有报错
5.为什么docker 比vm虚拟机快
 与宿主机共享os vm是在宿主机上os运行虚拟机的os
6. docker仅包含业务运行所需的runtime环境
7.Docker Compose是一个用于定义和运行多个Docker容器的工具。它使用一个单独的YAML文件来配置和管理多个容器的相关设置，以便将它们作为一个应用程序一起运行。
8.管道符 | 是一种用于将一个命令的输出作为另一个命令的输入的特殊符号
